LlullGen Conversational & Analytical Memory System

An eight-layer architecture that moves continuity from LLM-honored prose to typed, governed, auditable state. The ActiveAnalyticalState is the single source of truth; rules live in code; transitions are events; users can inspect, correct, confirm, and freeze. LLMs classify and narrate — they do not contain memory.

MemoryService Protocol

Single seam for all consumers

The only memory interface used outside memory/. Orchestrator, agents, and routes depend on the protocol — never on raw transcript or DB rows.

📌 get_active_state

Immutable snapshot of ActiveAnalyticalState for a session.

→ ActiveAnalyticalState (frozen)

🧮 propose_state_update

Returns a StateProposal with per-slot decisions.

→ StateProposal { followup_class, decisions[], pending[], grounded_tokens, sql_artifact }

✅ commit_state_update

Applies the proposal; emits MemoryEvents; bumps version.

→ StateCommitResult

🪟 get_short_range

Typed window over recent turns for prompts that genuinely need it.

→ ShortRangeView (max_turns, max_chars_per_turn)

🗃️ get_session_objects

ObjectBus catalog with grain contract — never raw DataFrames.

→ list[ObjectMetadata { grain, columns, source_query }]

🧬 get_previous_sql

Replaces direct plan_execution_summary JSONB queries.

→ Optional[SqlArtifact { sql, grain, generated_at_turn }]

📡 emit_event

Publishes a MemoryEvent to the audit stream.

slot_inherited | overwritten | cleared | clarification_* | user_*

🔍 read_audit

Replayable per-slot transition log per session.

→ list[StateTransition]
